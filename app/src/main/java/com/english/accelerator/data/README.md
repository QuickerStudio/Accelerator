# Data 层说明

## 职责范围

data 文件夹负责应用的数据持久化和数据访问，包含：

### 1. 数据存储和读取
- SharedPreferences 的封装和操作
- 数据的持久化存储
- 数据的读取和查询

### 2. 数据模型定义
- 定义应用中使用的数据结构
- 数据传输对象（DTO）
- 实体类（Entity）

### 3. 数据访问接口
- 提供统一的数据访问接口
- 对外隐藏具体的存储实现细节

## 设计原则

- **单一职责**：只负责数据的存储和读取，不包含业务逻辑
- **无算法逻辑**：不包含任何算法计算，算法逻辑应放在 algorithm 层
- **接口抽象**：通过接口向其他层提供数据访问能力
- **数据封装**：封装 SharedPreferences 等底层存储细节

## 与其他层的关系

- **algorithm 层**：通过 data 层提供的接口获取数据，不直接操作 SharedPreferences
- **UI 层**：通过 data 层读取和保存用户数据
- **业务层**：依赖 data 层进行数据持久化
