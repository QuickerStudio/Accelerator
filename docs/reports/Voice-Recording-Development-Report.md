# 语音录音功能研发报告

## 项目信息
- **功能名称**: 对话界面语音录音输入
- **开发日期**: 2026-03-02
- **状态**: ✅ 研发成功

## 功能需求
实现微信式的语音输入交互：
1. 点击输入框 → 文字输入模式
2. 长按输入框（≥500ms）→ 语音录音模式
3. 录音时背景变蓝色
4. 松开手指 → 发送语音消息

## 技术挑战

### 挑战1: 点击穿透问题
**问题**: 手势检测层覆盖在输入框上，阻止了正常的点击输入。

**尝试方案**:
1. ❌ 使用 TextField + 透明覆盖层 → 覆盖层阻挡点击
2. ❌ 条件显示手势层（`if (!isFocused)`）→ 第一次点击后层消失，无法再次长按
3. ✅ **最终方案**: 事件消费机制

### 挑战2: 长按触发系统键盘
**问题**: BasicTextField 的默认长按行为会触发文本选择和系统键盘。

**解决方案**:
- 长按时调用 `down.consume()` 和 `up.consume()` 消费事件
- 短按时不消费事件，让事件穿透到 TextField

### 挑战3: 录音状态管理
**问题**: 长按触发录音后，如何在同一个手势序列中检测松开事件。

**解决方案**:
```kotlin
awaitEachGesture {
    val down = awaitFirstDown(requireUnconsumed = false)
    val result = withTimeoutOrNull(500L) {
        waitForUpOrCancellation()
    }

    if (result == null) {
        // 长按 - 消费事件，启动录音
        down.consume()
        isRecording = true

        // 继续等待松开
        val up = waitForUpOrCancellation()
        if (up != null) {
            up.consume()
            isRecording = false
            onSend()  // 发送语音
        }
    } else {
        // 短按 - 不消费，穿透到 TextField
    }
}
```

## 最终实现方案

### 核心架构
1. **BasicTextField** - 无背景的自定义文本输入框
2. **背景容器** - 纯装饰层，提供圆角和背景色
3. **手势检测层** - 始终存在，覆盖在输入框上方

### 关键技术点
- `awaitFirstDown(requireUnconsumed = false)` - 允许检测已处理的事件
- `withTimeoutOrNull(500L)` - 500ms 长按阈值判断
- `consume()` - 事件消费机制，阻止事件传递
- 状态驱动 UI - `isRecording` 控制背景色变化

### 交互流程
```
用户按下
    ↓
等待 500ms
    ↓
├─ 500ms 内松开 → 短按 → 不消费事件 → TextField 获得焦点 → 文字输入
└─ 500ms 后仍按住 → 长按 → 消费事件 → 背景变蓝 → 录音模式
                                    ↓
                              等待松开
                                    ↓
                              松开 → 发送语音 → 恢复默认状态
```
